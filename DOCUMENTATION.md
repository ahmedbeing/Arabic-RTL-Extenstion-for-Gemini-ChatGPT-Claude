# دليل إضافة Arabic RTL for AI Chats

إضافة متصفح (Chrome / Edge / Brave) تضبط اتجاه النصوص العربية تلقائياً في
محادثات **Claude** و **ChatGPT** و **Gemini**، فتتحاذى الكتابة من اليمين لليسار
بشكل طبيعي بدل ما تطلع مكسّرة.

---

## 1) شرح الملفات ووظيفة كل واحد

```
Arabic RTL/
├── manifest.json
├── src/
│   ├── content.js
│   └── styles.css
├── popup/
│   ├── popup.html
│   ├── popup.css
│   └── popup.js
└── DOCUMENTATION.md  ← هذا الملف
```

### `manifest.json`
بطاقة هوية الإضافة. المتصفح يقرأ هذا الملف أول شيء ليعرف:
- اسم الإضافة ووصفها وإصدارها.
- على أي مواقع تشتغل (`host_permissions` و `matches`).
- وش الصلاحيات اللي تحتاجها (`storage` لحفظ الإعدادات، `activeTab` لإعادة
  تحميل التبويب من الـ popup).
- أي ملفات JS/CSS تُحقن في الصفحة (`content_scripts`).
- الواجهة المنبثقة عند الضغط على أيقونة الإضافة (`action.default_popup`).

تعدّل هنا الإصدار `version` كل مرة تحدّث الإضافة قبل ما ترفعها.

### `src/content.js`
العقل الفعلي للإضافة — JavaScript يُحقن في صفحات Claude/ChatGPT/Gemini.

- يحدّد الموقع الحالي ويختار المحدِّدات (selectors) المناسبة لحقل الإدخال
  والرسائل من كائن `SITES`.
- على حقل الإدخال: يستمع لحدث `input`، يقرأ أول حرف قوي في النص، ويضبط
  `dir="rtl"` لو الحرف عربي و `dir="ltr"` لو لاتيني.
- على الرسائل: يطبّق `dir="auto"` فيتولّى المتصفح اتجاه كل فقرة وحدها.
- يستعمل `MutationObserver` لمتابعة الرسائل الجديدة لأن المواقع SPA
  (single-page app) — الرسائل ما تتحمّل بـ refresh كامل.

### `src/styles.css`
تنسيقات CSS بسيطة ترافق `content.js`:
- تضمن محاذاة العناصر RTL لليمين.
- تحافظ على اتجاه الأكواد (`code`, `pre`) من اليسار لليمين حتى لو الفقرة
  العربية حواليها — ضروري عشان الكود ما ينقلب.

### `popup/popup.html`
نافذة الإعدادات اللي تطلع لما تضغط على أيقونة الإضافة في شريط المتصفح.
فيها:
- مفتاح تفعيل/إيقاف عام.
- مفاتيح منفصلة لكل موقع (Claude / ChatGPT / Gemini).
- زر إعادة تحميل التبويب الحالي بعد التعديل.

### `popup/popup.css`
تنسيق النافذة المنبثقة (ألوان، أزرار التبديل، التخطيط).

### `popup/popup.js`
يربط النافذة بـ `chrome.storage.sync` — يحفظ الإعدادات تلقائياً ويستعيدها
عند فتح الـ popup. الـ `content.js` يقرأ نفس الإعدادات قبل ما يشتغل.

## 2) التشغيل محلياً (Developer Mode)

هذي الطريقة لتجربة الإضافة على جهازك بدون نشر.

1. افتح المتصفح وروح للعنوان:
   - Chrome / Brave: `chrome://extensions`
   - Edge: `edge://extensions`
2. فعّل **Developer mode** من الزاوية (يمين أو يسار حسب اللغة).
3. اضغط **Load unpacked**.
4. اختر مجلد المشروع: `C:\Users\Amalg\repos\Arabic RTL`.
5. الإضافة بتظهر في القائمة. ثبّتها في شريط الأدوات (أيقونة الـ pin).
6. افتح [claude.ai](https://claude.ai) أو [chatgpt.com](https://chatgpt.com) أو
   [gemini.google.com](https://gemini.google.com) واكتب عربي — لازم النص يتحاذى
   لليمين تلقائياً.

### عند تعديل أي ملف
ارجع لـ `chrome://extensions` واضغط زر **🔄 Reload** على بطاقة الإضافة، بعدها
حدّث صفحة الموقع.

## 3) النشر على Chrome Web Store

### تنبيه عن السعر
رسوم تسجيل المطور في Chrome Web Store هي **5 دولارات (دفعة واحدة فقط، مدى
الحياة)**، مو دولار واحد. هذي رسوم لمرة واحدة على الحساب نفسه، تقدر بعدها
تنشر إضافات غير محدودة بدون رسوم إضافية.

### الخطوات

#### 1. جهّز الأيقونات
المتجر يطلب أيقونات بهذه المقاسات (PNG):
- `icon16.png` — 16×16
- `icon32.png` — 32×32
- `icon48.png` — 48×48
- `icon128.png` — 128×128

حطّها في مجلد `icons/` داخل المشروع، ثم رجّع قسم `icons` في `manifest.json`:

```json
"icons": {
  "16": "icons/icon16.png",
  "32": "icons/icon32.png",
  "48": "icons/icon48.png",
  "128": "icons/icon128.png"
},
"action": {
  "default_title": "Arabic RTL",
  "default_popup": "popup/popup.html",
  "default_icon": {
    "16": "icons/icon16.png",
    "32": "icons/icon32.png"
  }
}
```

#### 2. جهّز صور المتجر
- **شعار صغير**: 128×128 PNG (نفس الأيقونة).
- **لقطات شاشة**: 1280×800 أو 640×400 PNG/JPG (لقطة واحدة على الأقل، يفضّل 3-5
  تبيّن الإضافة شغّالة على المواقع الثلاثة).
- **صورة Promo (اختيارية)**: 440×280.

#### 3. اضغط ملفات الإضافة في ZIP
- اختر كل الملفات داخل المجلد (manifest.json + src/ + popup/ + icons/).
- **لا تضغط** المجلد الأب نفسه — اضغط محتوياته.
- لا تضمّن `.git`، `.claude`، `node_modules`، أو أي ملف تطوير.
- اسم الملف مثلاً: `arabic-rtl-v1.0.0.zip`.

#### 4. سجّل حساب مطوّر
1. روح لـ [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole).
2. سجّل دخول بحساب Google.
3. ادفع رسوم التسجيل **5$** مرة وحدة (تحتاج بطاقة فيزا/ماستر).

#### 5. ارفع الإضافة
1. من لوحة التحكم اضغط **New item**.
2. ارفع ملف الـ ZIP.
3. عبّي صفحة المتجر:
   - **Description**: وصف بالعربي والإنجليزي عن الإضافة.
   - **Category**: Productivity.
   - **Language**: Arabic (و English لو تبي تستهدف جمهور أوسع).
   - **Screenshots**: ارفع اللقطات.
   - **Privacy practices**: حدد إن الإضافة ما تجمع بيانات (المربع الأول
     يكفي — Single purpose: ضبط RTL).
   - **Justification للصلاحيات**:
     - `storage` → "حفظ تفضيلات المستخدم محلياً (تشغيل/إيقاف لكل موقع)".
     - `host_permissions` → "حقن سكربت ضبط الاتجاه على المواقع المدعومة فقط".
4. اختر **Visibility**: Public أو Unlisted (الأخير ما يظهر بالبحث، يوصل بالرابط فقط).
5. اضغط **Submit for review**.

#### 6. المراجعة
- تستغرق عادة من **يوم لأسبوع**.
- أحياناً قوقل ترفض وتطلب تعديلات (وصف أوضح، تبرير صلاحيات، إلخ) — تعدّل وترفع
  مرة ثانية بدون رسوم إضافية.

### نصائح للقبول من أول مرة
- اكتب **Privacy Policy** بسيطة ولو على GitHub Pages؛ خصوصاً لو حابب الإضافة
  Public. تكفي صفحة تقول: "هذه الإضافة لا تجمع ولا ترسل أي بيانات. كل
  الإعدادات محفوظة محلياً عبر `chrome.storage.sync`".
- خلّي وصف الـ Single Purpose واضح ومحدّد جداً — قوقل تعتبر الوصف الغامض
  سبب رفض.
- لا تطلب صلاحيات أكثر مما تحتاجه فعلاً (إحنا فعلاً مقتصدين بالصلاحيات حالياً).

---

## 4) تحديث الإضافة لاحقاً

1. عدّل الكود.
2. ارفع رقم `version` في `manifest.json` (مثلاً `1.0.0` → `1.0.1`).
3. اضغط ZIP جديد.
4. من Developer Dashboard → اختر الإضافة → **Package** → ارفع النسخة الجديدة
   → **Submit for review**.

التحديث عادة يخلص أسرع من النشر الأول.

---

## 5) استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| الإضافة ما تشتغل بعد التعديل | اضغط 🔄 Reload في `chrome://extensions` ثم حدّث الموقع |
| النص ما يتحاذى لليمين في موقع معيّن | افتح `src/content.js` وحدّث المحدِّدات في كائن `SITES` (المواقع تغيّر HTML) |
| الـ popup فاضي أو ما يفتح | افتح Developer Tools على الـ popup (right-click على الأيقونة → Inspect popup) وشوف الأخطاء |
| الكود البرمجي يطلع معكوس | تأكد من `styles.css` فيه قاعدة `code, pre { direction: ltr }` |
